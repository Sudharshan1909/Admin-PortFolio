# Admin Portfolio

A modern portfolio and admin dashboard built with Next.js, TypeScript, and Tailwind CSS. This project combines a public-facing portfolio website with an admin experience for managing profile content, portfolio sections, visual themes, and screen mode settings.

DEMO LINK:[https://admin-portfolio-emqn.onrender.com/](https://admin-portfolio-emqn.onrender.com/)

## Overview

This project is designed to solve two needs in one codebase:

- A polished public portfolio for showcasing profile details, experience, skills, projects, and certificates.
- A private admin dashboard for updating content and controlling presentation settings without editing every page manually.

The application is structured around reusable layouts, centralized site data, and customizable UI themes, making it easier to maintain and extend over time.

## Features

- Public portfolio homepage with professional profile presentation.
- Dedicated pages for career, experience, skills, projects, and certificates.
- Admin dashboard for managing portfolio content.
- Settings section for selecting public profile themes.
- Dark and light screen mode support.
- Centralized site data model for easier content updates.
- Responsive UI for desktop and mobile devices.
- Built with a scalable Next.js App Router structure.

### Prerequisites

Make sure the following tools are installed:

- Node.js 18 or later
- npm, pnpm, or yarn
- Git

### Installation

```bash
npm install
```

### Run the development server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Usage

### Public site

The public site is intended to present professional information in a clean and customizable format. It includes profile details, social links, and dedicated portfolio sections.

### Admin dashboard

The admin dashboard is used to manage the site content and visual preferences. From the settings area, you can switch between multiple public profile themes and choose dark or light screen mode.

## Configuration

Update content and settings from the project data layer.

Common areas to customize:

- Profile details
- Contact links
- Career and education entries
- Experience records
- Skills
- Projects
- Certificates
- Theme and mode preferences

If environment variables are added later, document them in a `.env.example` file.
